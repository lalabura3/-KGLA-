## 2. 测试分层策略

```
         ┌──────────┐
         │   E2E    │  核心用户路径（10-15 条）
         │  ~15min  │  Playwright, 5 browsers
         ├──────────┤
         │ 集成测试  │  全链路 API + DB + ASR
         │  ~5min   │  pytest-asyncio + httpx
         ├──────────┤
         │ 单元测试  │  覆盖率 ≥ 70%
         │  ~30s    │  pytest + Jest
         └──────────┘
```

### 2.1 单元测试

| 层级 | 框架 | 覆盖目标 | 运行时间 |
|------|------|---------|---------|
| 后端 (Python) | pytest + pytest-cov | ≥ 70% 行覆盖 | ≤ 30s |
| 前端 (TypeScript) | Jest + ts-jest | ≥ 70% 分支覆盖 | ≤ 30s |

**重点覆盖模块：**
- ASR Pipeline: 音频提取、VAD 分段、Whisper 调用、结果合并
- LLM Pipeline: Prompt 构建、输出解析、JSON Schema 校验、幻觉检测
- 知识图谱: 图构建、查询、序列化、性能边界
- 视频处理: 上传、格式校验、进度追踪、状态机

### 2.2 集成测试

| 测试场景 | 预计用例数 |
|---------|-----------|

| WebSocket 进度推送 | 3 |
| API 错误处理与状态码 | 8 |
| 数据库读写一致性 | 5 |

### 2.3 E2E 测试

| 浏览器 | 设备 | 用例数 |
|--------|------|--------|
| Chromium | Desktop 1920×1080 | 15 |
| Firefox | Desktop 1920×1080 | 15 |
| Safari (WebKit) | Desktop | 15 |
| iPad (gen 9) | 810×1080 | 10 |
| iPad Pro 12.9 | 1024×1366 | 10 |

**核心 E2E 场景：**
1. 新用户访问 → 看到引导页
2. 上传视频 → 进度条 → 处理完成 → 自动跳转
3. 查看 AI 笔记 → 点击概念 → 图谱聚焦
4. 点击时间戳 → 视频跳转到对应位置
5. 图谱搜索 → 匹配节点高亮
6. 图谱拖拽/缩放/聚焦 → 交互流畅
7. 错误处理：上传非视频文件 → 错误提示
8. 平板端：侧边栏折叠/展开、触屏手势

## 3. 功能验收标准

### 3.1 视频上传

| 编号 | 验收项 | 标准 | 优先级 |
|------|--------|------|--------|
| F-01 | 支持 MP4 格式上传 | 上传成功率 ≥ 99% | P0 |
| F-02 | 文件大小限制 | ≤ 5GB, 超限返回 413 + 清晰提示 | P0 |
| F-03 | 格式校验 | 非视频格式返回 415 + 错误信息 | P0 |
| F-04 | 上传进度显示 | 实时百分比 + 预计剩余时间 | P0 |
| F-05 | 断点续传 | 网络中断后可从断点恢复 | P1 |

### 3.2 ASR 语音识别

| 编号 | 验收项 | 标准 | 优先级 |
|------|--------|------|--------|
| A-01 | 处理时间 | 45分钟视频 ≤ 30分钟处理完毕 | P0 |
| A-02 | CER (标准普通话) | ≤ 5% | P0 |
| A-03 | CER (综合) | 整体 ≤ 8% | P0 |
| A-04 | 中英混合识别 | 英文术语正确率 ≥ 90% | P0 |
| A-05 | 时间戳对齐精度 | 误差 ≤ ±1 秒 | P0 |
| A-06 | 处理失败重试 | 自动重试 3 次后标记失败 | P0 |

### 3.3 AI 笔记生成

| 编号 | 验收项 | 标准 | 优先级 |
|------|--------|------|--------|
| N-01 | 笔记结构完整性 | 包含标题、摘要、关键概念、关系 | P0 |
| N-02 | 关键概念提取率 | 人工标注概念的召回率 ≥ 80% | P0 |
| N-03 | JSON 格式合规 | 100% 输出符合预定义 JSON Schema | P0 |
| N-04 | 幻觉检测 | 无凭空捏造的概念（基于原文验证） | P0 |
| N-05 | 笔记生成时间 | ≤ 视频时长的 50% | P0 |

### 3.4 知识图谱

| 编号 | 验收项 | 标准 | 优先级 |
|------|--------|------|--------|
| G-01 | 图谱正确渲染 | 所有节点和边可见 | P0 |
| G-02 | 节点点击详情 | 点击节点展示关联笔记和时间戳 | P0 |
| G-03 | 图谱交互性能 | 200 节点/300 边时帧率 ≥ 30fps | P0 |
| G-04 | 拖拽缩放流畅 | 操作延迟 ≤ 100ms | P0 |
| G-05 | 搜索高亮 | 搜索结果在 1 秒内高亮显示 | P1 |

### 3.5 系统可用性

| 编号 | 验收项 | 标准 | 优先级 |
|------|--------|------|--------|
| S-01 | API 可用性 | 核心 API 成功率 ≥ 99% | P0 |
| S-02 | 错误兜底 | 所有异常返回友好错误信息+重试入口 | P0 |

## 4. 性能验收标准

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| 45min 视频 ASR 处理时间 | ≤ 30min (P95) | 10 次独立运行取 P95 |
| LLM 笔记生成时间 | ≤ 视频时长 × 50% | 同上 |
| 图谱首次渲染 (100节点) | ≤ 2s | Chrome DevTools Performance |
| 图谱交互帧率 | ≥ 30fps (200节点) | FPS meter |
| API 响应时间 (P95) | ≤ 200ms (非视频处理) | 负载测试 100 RPS |
| 页面首次加载 (LCP) | ≤ 2.5s | Lighthouse |
| 首次输入延迟 (FID) | ≤ 100ms | Lighthouse |

## 5. 安全验收标准

| 编号 | 验收项 | 标准 |
|------|--------|------|
| SE-01 | 文件类型白名单 | 仅接受白名单视频格式 |

| SE-05 | CORS 配置 | 仅允许前端域名 |

## 8. Bug 严重等级定义

| 等级 | 定义 | 示例 |
|------|------|------|
| **P0 - 阻断** | 核心功能完全不可用 | 视频上传 100% 失败、ASR 崩溃 |
| **P1 - 严重** | 核心功能严重受损 | 图谱 50% 节点不可点击、笔记丢失关键内容 |
| **P2 - 一般** | 功能可用但有缺陷 | 某些字体渲染异常、时间戳偏差 2-3 秒 |
| **P3 - 轻微** | 体验优化项 | 加载动画不够流畅、文案需优化 |
