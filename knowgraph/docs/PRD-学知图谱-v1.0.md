## 1. 产品概述

### 1.1 愿景

让每一段视频学习过程变得可感知、可追溯、可交互。通过 AI 自动将视频内容结构化，生成可浏览的知识网络，帮助学习者从"看视频"升级为"构建知识体系"。

### 1.2 目标用户

| 用户类型 | 画像 | 核心需求 |
|---------|------|---------|
| 学生/自学者 | 在线学习网课、讲座、公开课 | 快速回顾视频重点，理清知识点关系 |
| 教师/讲师 | 上传教学视频进行备课或分享 | 查看视频自动生成的笔记和知识结构，用于教学评估 |
| 研究人员 | 观看学术讲座、技术会议录像 | 快速定位关键知识点，建立知识关联 |

### 1.3 核心价值主张

- **从看到懂：** AI 自动提取知识点，将线性视频转化为结构化知识网络
- **从找到查：** 每个知识点绑定视频时间戳，点击节点即可跳转到对应的视频位置
- **从散到连：** 展示知识点之间的关联关系，构建完整的知识图谱

## 2. 核心用户流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌────────────┐
│ 导入视频 │ →  │ AI 处理  │ →  │ 查看笔记 │ →  │ 探索图谱   │
│         │    │ ASR+LLM  │    │ 图文+戳  │    │ 节点+跳转  │
└──────────┘    └──────────┘    └──────────┘    └────────────┘
```

### 2.1 完整用户旅程

1. **导入阶段**：用户上传视频文件或输入公开视频链接
2. **处理阶段**：系统自动执行 ASR（语音识别）→ 分段 → LLM 摘要 → 知识点提取 → 关系构建
3. **学习阶段**：用户查看 AI 生成的图文并茂的笔记，每段笔记绑定时间戳
4. **探索阶段**：用户在知识图谱中浏览节点，点击节点查看详情并跳转对应视频位置
5. **深化阶段**（P1）：用户编辑笔记、搜索图谱、AI 问答、标记掌握程度

## 3. 功能需求（MVP v1.0）

### 3.1 P0 — 核心链路（必须完成）

| 功能模块 | 功能描述 | 验收标准 |
|---------|---------|---------|
| F001 视频上传 | 支持 .mp4/.mov/.avi/.mkv 格式上传，显示上传进度，文件大小限制 2GB | 上传 -> 显示进度条 -> 成功后进入处理流程 |
| F002 链接导入 | 支持输入视频公开链接（bilibili/YouTube 等），后端抓取或处理 | 粘贴链接 -> 自动识别 -> 进入处理流程 |
| F003 视频播放器 | 自定义 HTML5 播放器：播放/暂停、进度条拖拽、音量、全屏、键盘快捷键 | 参考 Alice 实现的 VideoPlayer 组件 |
| F004 ASR+分段 | Whisper 语音识别 + 自动分段（按语义停顿/句号切分），支持中文+英文 | 45 分钟视频在 30 分钟内完成 ASR 处理 |
| F005 AI 笔记 | LLM 为每段生成标题+摘要+关键帧截图（可选），图文混排展示 | 每段包含：标题、摘要文本、时间戳 |
| F006 笔记面板 | 左侧面板展示按时间顺序排列的笔记段落，点击时间戳跳转视频 | 点击段落 -> 视频跳转到对应时间点播放 |
| F007 知识图谱 | 单视频知识点图谱，节点=知识点，边=关系（前置/包含/相似/因果等） | 图谱正确渲染，节点可拖拽，物理模拟正常工作 |
| F008 节点详情 | 点击图谱节点弹出详情卡片：名称、描述、类型、重要度、关联时间戳 | 点击节点 -> 显示详情卡片 -> 点击时间戳跳转视频 |
| F009 图谱模式切换 | 章节聚类模式（默认展示章节聚类的图谱）→ 聚焦模式（局部展开）→ 路径模式（链式展示） | 三种模式可切换，过渡动画流畅 |
| F010 视频元数据提取 | 自动提取视频标题、时长、语言检测 | 视频进入时将元数据存入数据库 |
| F011 视频状态轮询 | 前端轮询 ASR/笔记/图谱处理进度，实时展示处理状态 | 状态变化时 UI 同步更新，处理完成自动跳转 |
| F012 学习记录列表 | 历史页面展示用户所有已处理的视频列表，支持搜索和分页 | 12 个/页，搜索防抖 300ms |

### 3.2 P1 — 增强功能（优先级高）

| 功能模块 | 功能描述 | 预期交付 |
|---------|---------|---------|
| F101 笔记编辑 | 用户可手动编辑/删除 AI 生成的笔记内容 | 富文本编辑器，支持 Markdown |
| F102 图谱搜索 | 在知识图谱中搜索知识点名称，高亮匹配节点 | 搜索框 + 实时过滤 |
| F103 AI 问答 | 基于视频内容回答用户提问，标注答案来源段落 | 聊天式界面，回答附带时间戳引用 |
| F104 掌握度标记 | 用户标记知识点的掌握程度（未学/学习中/已掌握），图谱节点用颜色区分 | 节点颜色变化，支持批量标记 |
| F105 学习仪表盘 | 概览页展示学习统计：视频数量、处理中的视频、知识点数、掌握进度 | 卡片式仪表盘，图表可视 |

### 3.3 非功能需求

| 维度 | 要求 |
|------|------|
| 响应式 | 支持 PC（1920+）、平板（768+），不额外适配手机 |
| 性能 | 图谱渲染 ≤ 200 节点无卡顿；页面首屏加载 ≤ 3s |
| 可访问性 | WCAG AA 级基线（语义标签、键盘导航、焦点可见、aria-live 区域） |
| 安全性 | ASR/LLM 处理完全在本地服务器完成，无数据外泄 |
| 错误处理 | 上传失败/处理失败/网络断开均有明确的错误提示和恢复路径 |
| 负载 | 支持 4×4090 服务器上 8-10 人同时使用 |

## 4. UI/UX 设计规范

### 4.1 交互参考

核心交互风格参考 **Obsidian Graph View**：

- 节点中心圆点+文字标签
- 物理引擎：节点互相排斥、边吸引，排列自然
- 拖拽交互：用户可拖拽任意节点调整布局
- 缩放交互：鼠标滚轮或触控板双指缩放

### 4.2 学习页面布局

```
┌─────────────────────────────────────────────────┐
│  Header (标题 + 返回 + 操作按钮)                  │
├────────────────┬─────────────────┬──────────────┤
│  笔记面板       │  视频播放器      │  图谱面板     │
│  (左侧 30%)     │  (中间 40%)     │  (右侧 30%)   │
│                │                  │              │
│  ┌─ 段落1 ───┐ │  [视频画面]      │  [知识图谱]   │
│  │ 00:12-03:45 │ │                  │  ○  ○  ○    │
│  │ 标题+摘要   │ │                  │   💠 ○ ○    │
│  └────────────┘ │                  │  ○  ○  💠 ○  │
│  ┌─ 段落2 ───┐ │  进度条 ██████░░ │  ─── ───    │
│  │ 03:45-08:20 │ │                  │              │
│  │ 标题+摘要   │ │  控制栏          │  [模式切换]   │
│  └────────────┘ │   ▶ ██ 🔊 ⛶     │  [节点详情]   │
└────────────────┴─────────────────┴──────────────┘
```

### 4.3 图谱交互模式

| 模式 | 触发 | 行为 |
|------|------|------|
| **章节聚类** | 默认 | 根据视频分段章节将知识点聚类显示，同章节节点距离更近 |
| **聚焦模式** | 点击节点 | 以选中节点为中心展开其 1-2 跳邻居节点，其余半透明 |
| **路径模式** | 选择两个节点 | 展示两节点之间的最短关联路径 |

### 4.4 视觉规范

- 颜色：蓝紫色系主色调（#0D8CE0），暖色（重要度高），冷色（重要度低）
- 节点大小：关联重要度（0.0-1.0），越大越重要
- 节点颜色：按知识类型区分（概念/术语/公式/方法/例子/人物/事件）
- 边线粗细：关系权重（0.0-1.0），越粗关联越强

## 6. 风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| ASR 处理超时（45min 视频 > 30min） | 中 | 高 | 第一周做性能验证；使用 faster-whisper/WhisperX 优化；失败按分段分发到多 GPU |
| LLM 提取知识点产生幻觉 | 中 | 高 | 每个知识点绑定至少一个视频时间戳用于溯源；卡片标注"AI 生成"；允许用户反馈 |
| 大图谱性能问题（>500 节点） | 中 | 中 | 默认只渲染局部子图（1-2 跳）；全局用 canvas 简化渲染 |
| 视频版权风险 | 低 | 高 | 仅处理用户上传/授权链接；不提供分发/下载；教育场景合理使用 |
| 用户上手困难 | 中 | 中 | 首次使用 3 步交互引导；内置示例视频 Demo；空状态友好提示 |
| 多用户并发导致 GPU OOM | 中 | 高 | 任务队列机制，单次最多处理 3 个视频；排队提示 |

## 9. 附录

### A. 数据模型（核心）

```
Video ---< NoteSegment (多个段落)
  |          |
  |          +--- timestamp, title, summary, content
  |
  +--- KnowledgeNode (多个节点)
  |        |
  |        +--- id, name, description, type, importance, timestamp
  |
  +--- Relation (多条关系边)
           |
           +--- source_node → target_node, relation_type, weight
```

### B. 词汇表

| 术语 | 说明 |
|------|------|
| ASR | Automatic Speech Recognition，自动语音识别 |
| Knowledge Graph | 知识图谱，由节点（知识点）和边（关系）构成的网络 |
| Segmentation | 视频按语义自动切分为段落 |
| Segment Index | 段落索引，对应视频中的时间区间 |
| Keyframe | 关键帧，为每个段落提取的代表性画面 |
| Mastery Level | 掌握度：未学/学习中/已掌握 |
